# 🏭 Initiator 階段二：腦內極限沙盤推演 (Mandatory Mental Simulation)

> 這是您作為高階架構師的最高價值。
在進入代碼生成前，**您必須在腦中模擬「平庸的 AI 工人」去執行上一階段 (Step 1) 拆解出來的任務時，會發生什麼災難。**

## 🛠️ 執行指南

請針對以下三個維度進行推演，並主動揪出盲點向人類報告修改：

### 1. 依賴衝突模擬
- 評估選擇的技術堆疊版本是否有已知的相容性問題。
- 這個前端框架跟這個後端路由會不會打架？
- 專案初始化的依賴是否會造成後續模組的依賴地獄？

### 2. 狀態機時序模擬
- 如果任務 A (資料庫建置) 被退回，任務 B (API 撰寫) 已經在 pending 等待，狀態的轉移會不會死鎖？
- 任務的先後順序是否完全合理且最小化等待時間？

### 3. 框架邊界條件 (Edge Cases)
- 檢視跨系統邊界：例如 Spring Boot WebSockets 對上 React Router，是否有跨域 (CORS) 漏抓？
- 若發生斷線，工人寫的 fallback 邏輯是否足夠穩健？

> 🛑 **暫停點 (Checkpoint)**：推演完畢並與使用者確認過修改後，請進入 **階段三：藍圖產製與建廠 (Factory Scaffolding)**。
